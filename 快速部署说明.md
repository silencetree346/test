# 快速部署到 GitHub Pages

## 由于本地缺少开发者工具，推荐直接部署到GitHub Pages

### 方法1：使用 GitHub Desktop（最简单）

1. **打开 GitHub Desktop**
   
2. **查看更改**
   - 左侧应该能看到 `docs/index.html` 等文件的更改

3. **提交更改**
   - 在左下角输入提交信息：`更新资讯获取功能`
   - 点击"Commit to main"

4. **推送到 GitHub**
   - 点击右上角的"Push origin"按钮

5. **启用 GitHub Pages**
   - 访问您的仓库页面（如：https://github.com/您的用户名/仓库名）
   - 点击 Settings（设置）
   - 左侧菜单找到 Pages
   - 在 Source 下选择：
     * Branch: `main`
     * Folder: `/docs`
   - 点击 Save

6. **访问网站**
   - 等待1-2分钟
   - 访问：`https://您的用户名.github.io/仓库名/`
   - 通过HTTPS访问，NewsAPI就能正常工作了！

### 方法2：临时先双击打开HTML（功能受限）

如果急需查看页面：
1. 直接双击 `/Users/luckincoffee/test/docs/index.html`
2. 虽然NewsAPI会报错，但页面基本功能可以看到
3. 等部署到GitHub Pages后再查看完整功能

### 为什么需要HTTP服务器或GitHub Pages？

- NewsAPI要求通过HTTP/HTTPS协议访问
- 直接打开HTML文件使用file://协议，会被CORS策略阻止
- GitHub Pages提供免费的HTTPS托管，完美解决这个问题
